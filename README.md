# Learning Git - the beginner's way, one step at a time
Learning Git from scratch - simple, practical, and beginner-friendly (learning in public), documented step by step.

I started this repository with one simple realization:
I was using GitHub… without actually understanding Git.

And apparently, that’s more common than I thought ;)

So this space is for learning Git properly — from scratch — in a way that actually makes sense.

Not rushed.
Just clear understanding, one step at a time.

## Why this repository exists

I have created this repository for everyone who:

* Does not want to randomly type git commands from a tutorial or an LLM
* Is tired of hoping nothing breaks
* Wants to know what’s actually happening behind the scenes

Instead of guessing and memorizing, the goal here is simple:
Let’s learn Git the right way — together.

## What this journey will cover

We’ll start from the absolute basics and gradually build up to more advanced concepts.

### Starting with the basics
- What version control actually means
- What Git is and why it exists
- Git vs GitHub (this confused me at first too)

### Getting comfortable with Git
- How Git stores changes behind the scenes
- Basic commands: init, add, commit, status, log

### Understanding branching
- What branches really are
- Creating and switching between them
- Merging and dealing with conflicts

### Working with GitHub
- Connecting a local project to GitHub
- push, pull, clone

### Learning to collaborate
- Pull requests
- Working with others on the same project

### Going a bit deeper
- Undoing changes safely
- Stashing work
- Using .gitignore

### Advanced concepts (once the basics feel natural)
- Rebase vs merge
- Keeping commit history clean
- A simple look into how Git works internally

## A small confession before we begin

This section is for all of us (including me) who once thought Git and GitHub were the same thing.

If that was you too, don’t worry. You’re not alone in this :)

This is a beginner-friendly journey.

So if something feels confusing at first, that’s okay.
We’re not rushing through this — we’re understanding it.

One concept at a time.

Think of it as: <br>
Learning first, documenting second.

I'm excited. Let's go!!!
